Coding Practices

  • arrow_drop_down
    Keeping Code Clean
    When coding it becomes very easy to cause an issue called "Spaghetti Code", this is when the code is organised poorly and due to this bad structure it can be daunting to maintain.
    It is important to keep code clean in all situations, it makes the code easier to mark in educational fields and in the work environment teams often work together making the code legible for the other team members is vital.
  • arrow_drop_down
    Commenting
    By adding comments to each section of the code, this can break down and seperate each section and provide content to search for with the Ctrl+F find tool found in most code editing software.
    Suitable comments include the name of the section or a quick explaination of how the code works if it is complicated, this will assist others trying to read the code comprehend it quicker.
  • arrow_drop_down
    Structure
    When structuring the code its important that the different sections have extra space between them as all coding languages ignore the extra spaces in function, this will assist it preventing "Spaghetti Code" as each section is quickly found while scrolling through.
    This extra spacing alongside other techniques such as commenting will assist anybody trying to read the code.
